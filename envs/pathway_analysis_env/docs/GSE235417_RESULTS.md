# GSE235417 (cetuximab resistance) — DGE + pathway enrichment

This report was generated by running `pathway_analysis_env` in **pipeline mode** on the public GEO study **GSE235417** (HNSCC PDX model UCLHN04; baseline vs acquired cetuximab-resistant; \(n=3\) per condition). Study page: `https://www.ncbi.nlm.nih.gov/geo/query/acc.cgi?acc=GSE235417`.

## What was run (inside OpenEnv)

- **Dataset**: `envs/pathway_analysis_env/data/geo/GSE235417_counts.csv.gz`
- **Groups**:
  - baseline: `ON68CTRL4`, `ON68CTRL5`, `ON68CTRL6`
  - resistant: `ON76Cx1`, `ON76CX3`, `ON76Cx4`
- **Contrast (DESeq2)**: `resistant` vs `baseline` (log2FC is resistant − baseline)
- **Prefilter**: genes with total counts across all samples < 10 removed
- **Differential expression**: **PyDESeq2** Wald test; significance threshold \(q \le 0.05\)
- **Pathway enrichment**: **over-representation analysis (ORA)** on DE-derived query genes using Enrichr libraries via `gseapy`:
  - `MSigDB_Hallmark_2020`
  - `KEGG_2021_Human`
  - `Reactome_2022`

Outputs saved:
- `envs/pathway_analysis_env/outputs/gse235417/summary.json`
- `envs/pathway_analysis_env/outputs/gse235417/de_top200.json`
- `envs/pathway_analysis_env/outputs/gse235417/enrichment_top50.json`
- Episode HTML trace: `envs/pathway_analysis_env/outputs/pathway_traces/gse235417-report.html` (path stored in `summary.json`)

## How to reproduce locally

From the repo root:

```bash
cd envs/pathway_analysis_env
uv sync
uv run - <<'PY'
from pathway_analysis_env.server.pathway_environment import PathwayEnvironment
from pathway_analysis_env.models import PathwayAction

env = PathwayEnvironment(case_file="gse235417_case.json")
env.reset()
env.step(PathwayAction(action_type="understand_experiment_design"))
de = env.step(PathwayAction(action_type="run_differential_expression"))
ora = env.step(PathwayAction(action_type="run_pathway_enrichment"))
print(de.top_genes[:10])
print(ora.top_pathways[:10])
PY
```

## Notes / interpretation guidance

- This is a **minimal, full-fidelity pipeline** (real DESeq2 + real ORA), but it is not yet a “publication-grade” workflow:
  - no explicit batch covariates (none provided here)
  - ORA uses Enrichr libraries (good for quick iteration, but you may prefer curated GMTs + GSEA for robustness)
  - the environment currently exports the **top 200 DE rows** for convenience; you can change that in `pathway_environment.py`
